* 차시: 10차시
* 작성자: 한성호
* 사용 파일: `sample_dco_log.txt`
* 사용 자료: 교육용 샘플 데이터

## 1. 분석 대상

* 날짜: 2026-07-03
* 샘플 장비명: `SAMPLE_TOR_SW_01`
* 인터페이스: `Gi0/1`
* Ticket ID: `EDU-TKT-2026-0003`
* 분석 시간: 03:05~03:30

## 2. 사건 전 상태

* [사실] 03:00에 `SAMPLE_TOR_SW_01`의 Normal heartbeat가 기록됐습니다.
* [사실] 사건 발생 전 시스템 상태는 정상으로 기록돼 있었습니다.

## 4. 로그에서 확인된 사실

* [사실] 03:05에 Gi0/1의 CRC Error가 5분 동안 154까지 증가했습니다.
* [사실] CRC Error 증가 1분 뒤 Gi0/1이 Down 상태로 변경됐습니다.
* [사실] Link Down과 함께 서버 연결 손실이 기록됐습니다.
* [사실] `EDU-TKT-2026-0003`이 생성됐습니다.
* [사실] 해당 Ticket이 Onsite Cabling Team으로 전달됐습니다.
* [사실] LC-LC 패치 코드 교체와 포트 커넥터 정리가 기록됐습니다.
* [사실] 조치 후 Gi0/1의 Link Up이 기록됐습니다.
* [사실] 03:30에 시스템 정상 상태와 CRC Error 0이 기록됐습니다.

## 5. 가능한 원인

### 가능성 1. 패치 코드 상태

* [추정] 패치 코드의 손상 또는 연결 상태가 사건과 관련됐을 가능성이 있습니다.
* 로그에서 찾은 근거: 패치 코드 교체 후 Link Up과 CRC Error 0이 기록됐습니다.
* 추가로 필요한 정보: 교체한 패치 코드의 실제 점검 결과와 손상 여부입니다.

### 가능성 2. 포트 커넥터 상태

* [추정] 포트 커넥터의 오염 또는 접촉 불량이 사건과 관련됐을 가능성이 있습니다.
* 로그에서 찾은 근거: 포트 커넥터 정리 후 연결이 복구됐습니다.
* 추가로 필요한 정보: 정리 전 커넥터의 오염과 접촉 상태입니다.

### 가능성 3. 포트 또는 반대편 인터페이스 상태

* [추정] 스위치 포트 또는 연결 대상 장비의 인터페이스 상태가 관련됐을 가능성이 있습니다.
* 로그에서 찾은 근거: CRC Error와 Link Down이 같은 Gi0/1에서 연속으로 기록됐습니다.
* 추가로 필요한 정보: 반대편 장비 로그와 Gi0/1의 이전 오류 기록입니다.

## 6. 현재 로그만으로 확인할 수 없는 내용

* [미확인] CRC Error가 Link Down의 유일한 원인인지는 알 수 없습니다.
* [미확인] 패치 코드와 커넥터 중 어느 부분이 직접적인 원인이었는지 알 수 없습니다.
* [미확인] 스위치 Gi0/1 포트 자체에 문제가 있었는지 알 수 없습니다.
* [미확인] 연결 대상 서버나 반대편 인터페이스에 문제가 있었는지 알 수 없습니다.
* [미확인] 패치 코드 교체 외에 다른 설정이나 작업이 있었는지 알 수 없습니다.

## 7. 추가로 확인하면 좋은 정보

1. CRC Error가 사건 이전에도 반복됐는지 확인합니다.
2. Gi0/1의 과거 상태와 오류 기록을 확인합니다.
3. 연결 대상 장비에도 같은 시간대 오류가 있었는지 확인합니다.
4. 패치 코드 교체 외에 다른 변경이나 작업이 있었는지 확인합니다.
5. 사건 이후 CRC Error가 다시 증가했는지 확인합니다.
6. Ticket에 추가 점검 결과가 기록됐는지 확인합니다.
