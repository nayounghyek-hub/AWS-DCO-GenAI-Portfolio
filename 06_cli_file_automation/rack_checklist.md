# [교육용 샘플 데이터 사용]

## EDU-SAMPLE-RACK-CHECKLIST

본 체크리스트는 실제 서버룸 또는 실제 랙(Rack)의 물리적/논리적 점검을 위한 것이 아닙니다. 
제공되는 교육용 샘플 로그 파일 및 샘플 티켓 정보를 분석하고 기록하기 위한 가이드라인입니다.

### 1. 로그 및 티켓 데이터 확인 단계
* [ ] **발생 시간 확인 (Event Timestamp)**
  - 샘플 로그와 티켓에 기록된 정확한 이벤트 발생 시각(예: KST 기준)을 찾아 기록했는가?
  - *기록란 (예시: EDU-YYYY-MM-DD HH:MM:SS)*: 
* [ ] **샘플 장비명 확인 (Sample Device ID)**
  - 문제가 발생한 샘플 장비명이 무엇인지 정확히 파악했는가? (예: `SAMPLE_TOR_SW_01` 등)
  - *기록란 (예시: SAMPLE_Device_Name)*: 
* [ ] **반복 여부 확인 (Recurrence Status)**
  - 동일한 에러(예: CRC 에러)가 특정 시간 동안 여러 번 반복해서 발생했는지 로그 내역을 비교 검토했는가?
  - *기록란 (예시: Single Event / Recurring Event)*:

### 2. 분석 및 보고 작성 시 주의사항
* [ ] **원인 단정 금지 (No Presumptions)**
  - 초기 샘플 데이터만으로 "케이블 불량", "장비 고장" 등 원인을 섣부르게 단정 짓지 않고, 가능한 기술적 시나리오를 나열하였는가?
